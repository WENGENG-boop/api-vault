FAQ

The practical questions, answered.

API Vault is intentionally straightforward about where it runs and what it can see.

Where does API Vault store data?

On the machine where you run it. Keys are encrypted in the local vault file, while app state and usage records remain local.

Which providers work with it?

It is built around OpenAI- and Anthropic-compatible request formats and can route to compatible upstream providers.

Can it see requests sent directly to a provider?

No. A request must pass through an API Vault proxy URL before it can be recorded.

Can I use it from another machine?

Yes, when you deliberately expose or deploy it through a secure reachable endpoint. Use proxy tokens instead of exposing real provider keys.
